Developer Guide

Technical documentation for contributors to HesabDari Danial.

1. Getting Started

Prerequisites

  • Node.js 18+ and npm
  • Git

Installation

git clone https://github.com/danialchoopan/DukanElectronJS.git
cd DukanElectronJS
npm install

Development

# Start dev server (Electron + Vite hot reload)
npm run dev

Testing

# Run all 57 tests across 5 suites
npm test

# Type check
npx tsc --noEmit

# Build for production
npx vite build

4. Data Architecture

4.1 Database

SQLite via better-sqlite3 in WAL mode. All tables defined in connection.ts with CREATE TABLE IF NOT EXISTS.

Key rule: All columns must be in CREATE TABLE. Only use ALTER TABLE inside migrateSchema() for auto-migration of old databases.

4.2 Repository Pattern

Each table has a dedicated repository file. Functions use getDatabase() directly:

// Example: products.ts
export function getAllProducts(): Product[] {
  const db = getDatabase()
  return db.prepare('SELECT * FROM products WHERE isActive = 1')
    .all()
    .map(formatProduct)
}

4.3 IPC Pattern

All UI↔DB communication goes through ipcMain.handle() in handlers.ts. UI calls via window.api (preload):

// main process (handlers.ts)
ipcMain.handle('products:getAll', () => products.getAllProducts())

// renderer (component)
const r = await window.api.products.getAll()
if (r.success && r.data) setProducts(r.data)

4.4 Double-Entry Accounting

Every financial transaction automatically posts to the general journal:

EventDebitCredit
Cash saleCash (1100)Sales Revenue (4100)
Credit saleAR (1400)Sales Revenue (4100)
PurchaseInventory (1300)AP (2110)
ExpenseExpense (6100-6700)Cash (1100)

5. How To Add

5.1 Adding a New View

  1. Create the component in renderer/src/views/YourView.tsx
  2. Add it to the view switch in App.tsx
  3. Add a navigation item to navConfig in the sidebar
  4. Add a translation key in i18n/

5.2 Adding a New IPC Handler

  1. Add the handler in src/main/ipc/handlers.ts
  2. Add the type signature in preload/index.ts
  3. Call from renderer via window.api.channelName()
// handlers.ts
ipcMain.handle('yourFeature:action', (event, arg) => {
  return yourRepository.doSomething(arg)
})

// preload/index.ts
yourFeature: {
  action: (arg: YourType) => ipcRenderer.invoke('yourFeature:action', arg)
}

5.3 Adding a New Repository

  1. Create src/main/database/repositories/yourTable.ts
  2. Define the table in connection.ts with CREATE TABLE IF NOT EXISTS
  3. Add expected columns to migrateSchema() for auto-migration
  4. Export CRUD functions

5.4 Adding a New Table Column

Important: When adding a new column, you must do TWO things:
  1. Add it to CREATE TABLE in connection.ts
  2. Add it to expectedColumns in migrateSchema() for auto-migration

8. Git Workflow

Branch Strategy

BranchPurpose
developActive development. All feature branches merge here.
productionStable releases. Only merge from develop when ready to release.
feature/*Individual feature branches. Merge into develop.

Commit Convention

Follow Conventional Commits:

<type>[scope]: <description>

# Examples:
feat(sales): add installment payment tracking
fix(accounting): correct trial balance calculation
docs: update developer guide

Release Process

  1. Merge develop into production
  2. Tag the release: git tag v1.0.0
  3. Push tag: git push origin v1.0.0
  4. Build and distribute
