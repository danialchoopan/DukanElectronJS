Features Guide

Complete reference for all features in HesabDari Danial — what each feature does, where to find it, and how it works.

3. Inventory Management

Products Tab

  • Searchable, sortable, filterable product list with pagination
  • Stock filters: All, In Stock, Low Stock, Out of Stock
  • Category filter dropdown
  • Restock panel: additive stock increase with reason
  • Low stock warning tiles at top

Report Tab

  • KPI cards: total products, purchase value, retail value, gross profit
  • Category donut chart (inventory value distribution)
  • Category bar chart (inventory value per category)
  • Category valuation table with inline progress bars
  • Slow-moving items (not sold in 30+ days)
  • A4 print and Excel export

Analytics Tab

  • Stock Status Donut: Healthy / Low / Out-of-stock products with counts
  • Stock Distribution Histogram: Products grouped by stock quantity ranges (0, 1-10, 11-30, etc.)
  • Category Value Chart: Bar or donut (toggleable) showing inventory value per category
  • Top 10 Products: Horizontal bars ranked by inventory value
  • Price Comparison: Side-by-side purchase vs sale price per category
  • KPI Cards: Total items, sale value, cost value, potential profit (all in Iranian format)

Adjustment Dialog

  • Search and select a product
  • Set absolute stock quantity (corrects actual count)
  • Choose adjustment type: manual, reconciliation, damage, count, other
  • Add reason and optional back-date
  • View adjustment history with before/after/delta

Audit Tab

  • Audit log of all system operations
  • Filter by action type (create, update, delete, restock, return)
  • Date range filtering

5. Customers

  • CRUD: Create/edit customers with type (real/legal), address, notes, image
  • Ledger: View all transactions (charges, payments, sales) with running balance
  • Charge/Pay: Add debt or record payments with accounting journal sync
  • Purchase History: Complete list of all purchases by customer
  • Delete Guard: Cannot delete customer with outstanding balance
  • Stats: Total spent, purchase count, last purchase date

6. Suppliers

  • CRUD: Create/edit suppliers with company info
  • Purchases: Record purchases with items, quantities, unit costs
  • Payments: Partial or full payment to suppliers with journal entry
  • Purchase Returns: Return items, restore stock, adjust supplier balance
  • Ledger: View all supplier transactions with balance
  • Accounting Sync: Purchases debit inventory, credit supplier payable

7. Backup & Export/Import

Auto Backup

  • Toggle on/off in Settings > Backup
  • Interval: daily, weekly, or monthly
  • Runs on app startup if enabled

Export (3-Step Dialog)

  1. Location: Local backups/ folder or custom path via system save dialog
  2. Format: SQLite (.db) — fast binary copy, or JSON (.json) — human-readable with metadata
  3. Scope: All data, Structure only (CREATE TABLE), or Selective (checkbox table picker with row counts)

Import

  1. Select file via system dialog
  2. Preview: version, timestamp, table count, row count
  3. Auto-creates pre-restore backup before importing

Database Reset

  1. Step 1: Save backup to custom location (mandatory before delete)
  2. Step 2: Final confirmation with warning of all data to be deleted
  3. App reloads with fresh empty database

9. Global Search

  • Full-text search across products, customers, suppliers
  • Search by name, barcode, category, phone number
  • Click result to navigate directly to the item (product detail, customer ledger, etc.)
  • Keyboard shortcut: Ctrl+K or the search button in sidebar

10. Setup Wizard

  • Appears on first run only
  • Step 1: Language, theme, store name/address/phone
  • Step 2: Admin account (name + 4-digit PIN)
  • Step 3: Business type selection (13 types) with auto-configured defaults
  • Restore option: Import from previous backup during setup

11. Date Handling

  • All display dates use Jalali (Shamsi) calendar via jalali.ts
  • Date inputs use ShamsiDateInput component (Jalali day/month/year picker)
  • Backdated invoices use HTML date input (Gregorian) which is converted to Jalali for display
  • Report dates: filterable by Jalali date range

12. Price Formatting

  • Iranian number format: formatPriceFA() → "10 million", "2.5 billion", "150 thousand"
  • Comma separators: formatPriceComma() → "10,000,000"
  • Inline editing: FormattedPriceInput component with live comma formatting
  • Chart labels: all use Iranian format

13. Charts & Visualization

  • Pure SVG: All charts built with inline SVG — no charting library dependency
  • 4 Color Themes: Default, Ocean, Forest, Warm — switchable at runtime
  • Text Size: Small / Medium / Large controls all chart labels
  • Chart Type Toggle: Bar ↔ Donut for category/expense charts
  • PNG Export: Download chart as image via SVG-to-canvas conversion
  • No Data State: Clean empty state with icon and message when data is absent

14. QR & Barcode System

  • QR Generation: Proper ISO 18004 QR codes with Reed-Solomon error correction
  • Barcode Input: Text input with autocomplete + camera scanning
  • Camera Scanner: html5-qrcode library — supports both QR codes and traditional barcodes (EAN-13, UPC-A, Code128)
  • Label Printing: Generate QR/barcode labels for products

15. Professional Calculator

  • Basic Arithmetic: +, -, ×, ÷ with keyboard shortcuts
  • Scientific Ops: √, x², 1/x, ±, %
  • Memory Functions: M+, M-, MR, MC with visual indicator
  • Currency Conversion: USD ↔ EUR ↔ IRR with static rates
  • History: Timestamped calculation history, click to re-use, copy all
  • Floating Mode: Bottom-right overlay + docked mode via sidebar
  • Global Shortcut: Ctrl+M toggles floating calculator from any screen

16. Role-Based Access Control (RBAC)

  • 7 Roles: Admin, Manager, Accountant, Salesperson, Warehouse, Cashier, Viewer
  • 35 Granular Permissions: sales:create, inventory:edit, accounting:journal, etc.
  • Permission Groups: Sales, Inventory, Accounting, Customers, Reports, Settings, Users
  • Utility Functions: hasPermission(), hasAnyPermission(), getEffectivePermissions()
  • User Management: Role assignment, permission configuration, login/activity tracking

17. Warranty & Repair Service

  • Service Tickets: Full workflow: received → diagnosing → awaiting_parts → in_repair → completed → returned
  • Parts Management: Add/remove parts per ticket with cost tracking
  • Warranty Tracking: Serial numbers, start/end dates, expiry alerts
  • Technician Assignment: Assign technicians to tickets
  • Cost Tracking: Parts + labor + shipping = total cost
  • History: Complete status change log per ticket

18. Customer Credit Management

  • Credit Limits: Per-customer maximum debt setting
  • Block/Unblock: Block types (credit, fraud, inactive, other) with reason tracking
  • Credit Score: 0-100 scale based on payment history, debt ratio, delay days
  • Real-time Check: Credit validation before each sale
  • Unblock Workflow: Customer requests → admin approves
  • Audit Trail: Every change logged in credit_history table

19. Audit Log & Restore Points

  • Enhanced Audit: userName, beforeValue, afterValue, IP tracking with search/filters
  • Auto-cleanup: Configurable retention (30d, 90d, 6mo, 1yr, 2yr) — toggleable on/off
  • Restore Points: Create named snapshots with SHA-256 verification
  • Point-in-Time Recovery: Create/verify/delete/restore from any saved point
  • Auto-cleanup: Keep last N points (5/10/20/50) — toggleable on/off

20. Database Migration System

  • Versioned Migrations: v1.0 → v1.5 with incremental scripts
  • Auto-detection: Detects current schema version on startup
  • Pre-migration Backup: Auto-creates backup before any migration
  • Migration History: All migrations logged with status (applied/failed)
  • Dry-run: Preview changes without applying
  • Validation: Post-migration integrity + foreign key + table checks
  • IPC + Preload: Schema version, dry-run, run, history, validate APIs
