۲. معماری داده‌ها

۲.۱ پایگاه داده

SQLite با better-sqlite3 در حالت WAL. تمام جداول در connection.ts تعریف شده‌اند.

نکته مهم: تمام ستون‌ها باید در CREATE TABLE باشند. از ALTER TABLE فقط در migrateSchema() برای مهاجرت خودکار استفاده می‌شود.

۲.۲ الگوی Repository

هر جدول یک فایل repository جداگانه دارد. توابع CRUD به صورت مستقیم از getDatabase() استفاده می‌کنند:

// مثال: products.ts
export function getAllProducts(): Product[] {
  const db = getDatabase()
  return db.prepare('SELECT * FROM products WHERE isActive = 1').all().map(formatProduct)
}

۲.۳ الگوی IPC

تمام ارتباط UI با دیتابیس از طریق ipcMain.handle() در handlers.ts انجام می‌شود. UI از طریق window.api (preload) فراخوانی می‌کند:

// main process (handlers.ts)
ipcMain.handle('products:getAll', () => products.getAllProducts())

// renderer (component)
const r = await window.api.products.getAll()
if (r.success && r.data) setProducts(r.data)

۳. مهاجرت خودکار

تابع migrateSchema() در connection.ts هنگام شروع برنامه اجرا می‌شود:

// 1. PRAGMA table_info() — ستون‌های موجود را می‌خواند
// 2. مقایسه با لیست ستون‌های مورد انتظار
// 3. ALTER TABLE ADD COLUMN برای ستون‌های جدید
// 4. ایدامپوتنت — پس از اولین اجرا no-op می‌شود
هنگام اضافه کردن ستون جدید: ستون را به لیست expectedColumns در migrateSchema() اضافه کنید و همچنین در CREATE TABLE قرار دهید.

۷. راهنمای اضافه کردن قابلیت جدید

  1. ستون‌های جدید را در CREATE TABLE اضافه کنید (connection.ts)
  2. ستون را در migrateSchema() برای مهاجرت خودکار اضافه کنید
  3. تایپ را در types/index.ts به‌روز کنید
  4. تابع repository مربوطه را به‌روز کنید (formatProduct, create, update)
  5. هندلر IPC را در handlers.ts اضافه کنید
  6. متد preload را در preload/index.ts اضافه کنید
  7. کامپوننت UI را بسازید یا به‌روز کنید
  8. تست: npx tsc --noEmit + node scripts/verify.js
