Backup & Migration

1. Backup System

Backups are managed by src/main/database/backup.ts. Two formats are supported:

FormatFileBest For
SQLite.db + .meta.jsonFast restore, device transfer
JSON.jsonReading, editing, version migration

Backup Metadata (.meta.json)

{
  "name": "auto-20260619",
  "hash": "sha256...",
  "size": 524288,
  "timestamp": "2026-06-19T12:00:00Z",
  "appVersion": "1.0.0",
  "tables": { "products": 32, "sales": 150 }
}

Export Options (3-step dialog)

  1. Location: Local backups/ folder or custom path via save dialog
  2. Format: SQLite (.db) or JSON (.json)
  3. Scope: All data / Structure only (CREATE TABLE) / Selective (checkbox table picker with row counts)

JSON Export Structure

{
  "version": "1.0.0",
  "appName": "hesabdari-danial",
  "format": "json" | "structure",
  "exportDate": "2026-06-19T...",
  "tables": { "products": [...], "sales": [...] }
  // OR
  "statements": ["CREATE TABLE ...", "CREATE TABLE ..."]
}

Restore Flow

  1. User selects file via system dialog
  2. Preview shows: version, timestamp, table count, row count
  3. On confirm: auto-creates pre-restore backup
  4. Copies backup .db over live database
  5. App reloads to apply

2. Auto-Migration

migrateSchema() in connection.ts runs after every getDatabase() call.

How It Works

// 1. Define expected columns per table
const expectedColumns = {
  products: [
    { name: 'subcategory', type: 'TEXT', defaultValue: "''" },
    { name: 'isSellable', type: 'INTEGER', defaultValue: '1' },
  ],
  sales: [
    { name: 'saleDate', type: 'TEXT', defaultValue: "datetime('now','localtime')" },
    { name: 'affectsInventory', type: 'INTEGER', defaultValue: '1' },
  ],
}

// 2. For each table, check existing columns
const existing = db.prepare('PRAGMA table_info(products)').all()
const existingNames = new Set(existing.map(c => c.name))

// 3. Add missing columns
if (!existingNames.has('subcategory')) {
  db.exec("ALTER TABLE products ADD COLUMN subcategory TEXT DEFAULT ''")
}

When to Add a New Column

Required steps:
  1. Add column to CREATE TABLE in connection.ts
  2. Add to migrateSchema() expectedColumns map
  3. Update formatProduct() in the repository with row.col ?? default
  4. Update TypeScript type in types/index.ts

3. Data-Level Migration (Smart Import)

For importing data from different versions, src/main/database/migration.ts provides:

  • detectFileVersion(data) — heuristic version detection
  • checkCompatibility(data) — returns migration steps needed
  • migrateData(data) — transforms row-level data between versions

This is separate from schema migration — it transforms the data payload, not the database structure.
