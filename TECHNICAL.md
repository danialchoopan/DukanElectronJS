## معماری سیستم

### مسیر پایگاه داده
| سیستم‌عامل | مسیر |
|-----------|------|
| Windows | `%APPDATA%/hesabdari-danial/pos.db` |
| macOS | `~/Library/Application Support/hesabdari-danial/pos.db` |
| Linux | `~/.config/hesabdari-danial/pos.db` |

پشتیبان‌ها در پوشه `backups/` کنار فایل اصلی ذخیره می‌شوند.

### نصب مجدد (Fresh Install)
1. فایل `pos.db` را از مسیر بالا حذف کنید
2. پوشه `backups/` را حذف کنید (اختیاری)
3. برنامه را مجدداً اجرا کنید — دیتابیس جدید با ساختار کامل ایجاد می‌شود
4. یا از دکمه «حذف کامل دیتابیس» در بخش تنظیمات > پشتیبان‌گیری استفاده کنید (با پشتیبان‌گیری خودکار قبل از حذف)

### پایگاه داده — جداول اصلی

| جدول | توضیح |
|------|-------|
| `users` | کاربران سیستم (admin/cashier) |
| `products` | کالاها با بارکد، قیمت، موجودی، دسته/زیردسته، وضعیت فروش |
| `customers` | مشتریان با حساب دفتری (حقیقی/حقوقی) |
| `suppliers` | تأمین‌کنندگان با حساب دفتری |
| `sales` + `sale_items` | فاکتورهای فروش با تاریخ دلخواه و نوع فروش |
| `purchases` + `purchase_items` | فاکتورهای خرید |
| `customer_ledger` + `supplier_ledger` | دفتر کل مشتریان/تأمین‌کنندگان |
| `categories` | دسته‌بندی سلسله‌مراتبی (۲ سطح) |
| `accounts` | دفتر حسابها (حسابداری دوطرفه) |
| `journal_entries` + `journal_entry_lines` | اسناد حسابداری |
| `fiscal_periods` | دوره‌های مالی |
| `expenses` | هزینه‌ها |
| `returns` | مرجوعی‌ها |
| `settings` | تنظیمات کلید-مقدار |
| `audit_log` | لاگ عملیات |
| `cash_register` | صندوق نقدی |
| `price_history` | تاریخچه تغییرات قیمت |
| `inventory_adjustments` | تاریخچه اصلاحات موجودی |

### ساختار محصولات
- **دسته‌بندی (category)**: متن آزاد — مطابق نام در جدول categories
- **زیردسته (subcategory)**: متن آزاد — فیلتر شده بر اساس دسته والد
- **وضعیت فروش (isSellable)**: اگر 0 باشد کالا در POS نمایش داده نمی‌شود
- **فعال (isActive)**: حذف نرم — کالا از لیست حذف می‌شود ولی در تاریخچه می‌ماند

### فاکتور فروش
- **تاریخ فاکتور (saleDate)**: قابل تنظیم توسط کاربر — امکان ثبت فاکتورهای تاریخ گذشته
- **تأثیر بر موجودی (affectsInventory)**: اگر غیرفعال باشد فقط مستندسازی است — موجودی تغییر نمی‌کند
- **شماره فاکتور**: از تاریخ فاکتور تولید می‌شود (INV-YYYYMMDD-NNNN)
- **نوع فروش**: حضوری / آنلاین

### اصلاح موجودی
- جدول `inventory_adjustments` تمام اصلاحات موجودی را با تاریخچه ثبت می‌کند
- انواع اصلاح: دستی، تطبیق با شمارش فیزیکی، ضایعات، شمارش دوره‌ای، سایر
- قابل ثبت با تاریخ دلخواه (برای تطبیق اسناد قبلی)

### همگام‌سازی خودکار با دفتر کل

| رویداد | ثبت حسابداری |
|--------|-------------|
| فروش نقدی | بدهکار صندوق، بستانکار فروش + بدهکار بهای تمام شده، بستانکار موجودی |
| فروش اعتباری | بدهکار حساب دریافتنی، بستانکار فروش |
| خرید از تأمین‌کننده | بدهکار موجودی، بستانکار حساب پرداختنی |
| پرداخت به تأمین‌کننده | بدهکار حساب پرداختنی، بستانکار بانک/صندوق |
| مرجوعی فروش | بدهکار فروش، بستانکار صندوق + بدهکار موجودی، بستانکار بهای تمام شده |
| ثبت هزینه | بدهکار حساب هزینه، بستانکار صندوق |

### پشتیبان‌گیری
- پشتیبان‌گیری خودکار با قابلیت تنظیم فاصله زمانی
- بازیابی با پیش‌نمایش و اعتبارسنجی نسخه
- حذف پشتیبان‌های انتخاب شده با تأیید
- **حذف کامل دیتابیس**: دکمه در تنظیمات > پشتیبان‌گیری — ابتدا پشتیبان ذخیره می‌شود، سپس تأیید نهایی

### نکات فنی مهم
- تمام ستون‌ها در تعریف `CREATE TABLE` هستند (بدون `ALTER TABLE`)
- Seed داده‌ها غیرفعال شده (کامنت شده در `index.ts` و `handlers.ts`)
- تمام تاریخ‌ها از طریق `jalali.ts` فرمت‌بندی می‌شوند
- فرمت قیمت با جداکننده هزارگان: `10,000,000`
- حذف دیتابیس از طریق IPC `database:reset` انجام می‌شود
