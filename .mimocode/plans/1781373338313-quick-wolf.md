## Step 2: Database Schema (`src/main/database/connection.ts`)

4 new tables:

```sql
accounts (id, code UNIQUE, name, type, parentId FK, isActive, description, createdAt)
  INDEX on type, code

fiscal_periods (id, name, startDate, endDate, isClosed, closedAt, closedBy FK, createdAt)

journal_entries (id, entryDate, description, referenceType, referenceId, fiscalPeriodId FK, isPosted, createdBy FK, createdAt)
  INDEX on entryDate, (referenceType+referenceId), fiscalPeriodId

journal_entry_lines (id, entryId FK CASCADE, accountId FK, debit, credit, description)
  INDEX on entryId, accountId
```

Seed default accounts (18 accounts for supermarket):
- 1000-1400 Assets (cash, bank, inventory, AR)
- 2000-2100 Liabilities (AP)
- 3000-3200 Equity (capital, retained earnings)
- 4000-4300 Income (sales, services, discounts)
- 5000-5100 COGS
- 6000-6700 Operating Expenses (rent, utilities, wages, supplies, repairs, transport, other)

Add tables to backup/restore array.

## Step 3: Repositories

- `getAllAccounts()`, `getAccountsByType(type)`, `getAccountById(id)`, `getAccountByCode(code)`

## Step 9: Migration Strategy

`accounting:migrate` handler:
1. Seed chart of accounts if empty
2. For each existing sale: generate journal entries (Dr cash/AR, Cr revenue + Dr COGS, Cr inventory)
3. For each existing expense: generate journal entry (Dr expense, Cr cash)
4. For each existing return: generate reverse journal entries
