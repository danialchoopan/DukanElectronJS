# Plan: Fix Product Add Bug + Soft Delete + Accounting Features

## Bug: Products not showing after add

### Root Causes Found
1. **Auto-barcode uses COUNT(*)** on ALL products (including deleted ones) → could cause UNIQUE constraint errors silently
2. **PopularItems/LooseItemsGrid never refresh** after mount (`useEffect([])`) — new products won't appear until remount
3. **No cross-view sync** — adding a product on one page doesn't update Inventory or POS views

### Fixes

#### 1. Fix auto-barcode in `src/main/ipc/handlers.ts`
- Change `SELECT COUNT(*) as c FROM products` to `SELECT MAX(id) as maxId FROM products`
- Use `(maxId || 0) + 1` instead of `count + 1`
- This avoids gaps from deleted products causing UNIQUE conflicts

#### 2. Add refresh mechanism
- After creating a product in AddProduct, emit `products:refresh` IPC event
- PopularItems and LooseItemsGrid listen for this event and re-fetch
- Simple and clean — no Zustand store needed

#### 3. Add soft delete with warning
- In AddProduct edit form, add "حذف" (Delete) button
- Confirmation dialog: "آیا از حذف این محصول اطمینان دارید؟ اطلاعات حسابداری حفظ می‌شود."
- Soft delete via `products:delete` (already sets `isActive = 0`)
- Product disappears from Inventory and POS but accounting data preserved

#### 4. Add delete button in Inventory
- Each row already has "ویرایش" (Edit) button
- Add "حذف" (Delete) button next to it with confirmation

### Files to modify
