# Plan: Advanced Inventory Management

## Features to Implement (from inventoryManger2.txt)

### 1. Audit Trail (Priority: HIGH)
Track all inventory changes with who, when, what.

**Database:** Add `audit_log` table
```sql
CREATE TABLE audit_log (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  userId INTEGER,
  action TEXT NOT NULL,  -- 'create', 'update', 'delete', 'restock', 'sale', 'return'
  entityType TEXT NOT NULL,  -- 'product', 'sale', 'expense'
  entityId INTEGER,
  details TEXT,  -- JSON with old/new values
  createdAt TEXT DEFAULT (datetime('now', 'localtime'))
);
```

**Files to modify:**
- `src/main/database/connection.ts` — add audit_log table
- `src/main/database/repositories/audit.ts` — new repository for audit log
- `src/main/ipc/handlers.ts` — log all inventory actions
- `src/renderer/src/views/Inventory.tsx` — show audit log in detail panel

### 2. Return Management (Priority: HIGH)
Handle customer returns with restocking and financial adjustment.

**Database:** Add `returns` table
```sql
CREATE TABLE returns (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  saleId INTEGER,
  userId INTEGER,
  productId INTEGER,
  quantity INTEGER,
  reason TEXT,
  refundAmount REAL,
  status TEXT DEFAULT 'pending',  -- 'pending', 'approved', 'completed'
  createdAt TEXT DEFAULT (datetime('now', 'localtime')),
  FOREIGN KEY (saleId) REFERENCES sales(id),
  FOREIGN KEY (userId) REFERENCES users(id),
  FOREIGN KEY (productId) REFERENCES products(id)
);
```

**Files to modify:**
- `src/main/database/connection.ts` — add returns table
- `src/main/database/repositories/returns.ts` — new repository
- `src/main/ipc/handlers.ts` — add return handlers
- `src/preload/index.ts` — add return API
- `src/renderer/src/views/SalesHistory.tsx` — return button already exists, need to implement
- `src/renderer/src/views/Dashboard.tsx` — show returns in stats
