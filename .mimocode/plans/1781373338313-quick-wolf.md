# Plan: Fix "Cannot find module '../database/connection'" Error

## Root Cause Found

**File:** `src/main/ipc/handlers.ts` line 47

```typescript
const db = require('../database/connection').getDatabase()
```

This `require()` call uses a relative path that resolves correctly in the source TypeScript but **fails in the compiled output** at `dist-electron/main/index.js` because the compiled file is a single bundled file that doesn't have the `../database/connection` module at that relative path.

The `getDatabase` function is **already imported** at line 12:
```typescript
import { isFirstRun, getDatabase } from '../database/connection'
```

**Fix:** Replace the `require()` call with the already-imported `getDatabase`:

```typescript
// BEFORE (broken):
const db = require('../database/connection').getDatabase()

// AFTER (fixed):
const db = getDatabase()
```

**File to modify:** `src/main/ipc/handlers.ts` line 47

This is a one-line fix. The `require()` was a leftover from when I added the auto-barcode logic and should have used the already-imported function.
