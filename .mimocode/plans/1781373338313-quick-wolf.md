# Plan: Redesign Categories System

## Database Changes (`connection.ts`)

Add columns to `categories` table:
```sql
ALTER TABLE categories ADD COLUMN slug TEXT DEFAULT '';
ALTER TABLE categories ADD COLUMN level INTEGER NOT NULL DEFAULT 0;
ALTER TABLE categories ADD COLUMN is_active INTEGER NOT NULL DEFAULT 1;
```

Use try/catch migration (same pattern as expenses.imageBase64).

Auto-populate `slug` from `name` (lowercase, dash-separated) and `level` (0 for parent, 1 for child) on existing rows.

## Repository Changes (`categories.ts`)

- Add `slug`, `level`, `isActive` to `Category` interface
- `createCategory`: auto-generate slug from name, set level based on parent
- `getCategoryTree`: return full hierarchy with product counts, level, isActive
- `deleteCategory`: **prevent** if category has products (query product count first)
- `getCategoryWithDescendants(id)`: return category + all descendant IDs (for filtering)
- `toggleCategoryActive(id)`: flip is_active
- `updateCategory(id, data)`: accept name, slug, isActive

## Product Filtering

In `products.ts`, update `getAllProducts` and `searchProducts` to support category hierarchy filtering:
- When filtering by a parent category, also include products in all child categories
- Add a new IPC handler `categories:descendants` that returns all child IDs for a given parent
