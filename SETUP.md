# راهنمای راه‌اندازی حسابداری دانیال

> **فارسی | [English](SETUP-en.md)**

## پیش‌نیازها

- Node.js 18+
- npm 9+
- Windows 10+ / macOS 12+ / Linux

## نصب

```bash
git clone https://github.com/danialchoopan/DukanElectronJS.git
cd DukanElectronJS
npm install
```

## اجرا

```bash
npm start
```

## ساخت نسخه اجرایی

### Windows
```bash
npx electron-builder --win
```
فایل اجرایی در `release/win-unpacked/` ایجاد می‌شود.

### macOS
```bash
npx electron-builder --mac
```

### Linux
```bash
npx electron-builder --linux
```

## مسیر پایگاه داده

| سیستم‌عامل | مسیر |
|-----------|------|
| **Windows** | `%APPDATA%\hesabdari-danial\pos.db` |
| **macOS** | `~/Library/Application Support/hesabdari-danial/pos.db` |
| **Linux** | `~/.config/hesabdari-danial/pos.db` |

پشتیبان‌ها در پوشه `backups/` کنار فایل اصلی ذخیره می‌شوند.

## ویزارد راه‌اندازی

اولین بار که برنامه را اجرا می‌کنید، ویزارد راه‌اندازی نمایش داده می‌شود:

### مرحله ۱: تنظیمات اولیه
- انتخاب زبان (فارسی/انگلیسی)
- انتخاب تم (تاریک/روشن)
- نام فروشگاه/کسب‌وکار
- آدرس و تلفن

### مرحله ۲: حساب مدیر
- نام مدیر
- رمز عبور ۴ رقمی

### مرحله ۳: تنظیمات مالی
- **انتخاب نوع کسب‌وکار** از ۱۳ گزینه
- تنظیمات پیش‌فرض خودکار بر اساس نوع کسب‌وکار
- فعال/غیرفعال کردن مالیات

## بازیابی اطلاعات قبلی

1. در ویزارد راه‌اندازی، "آیا اطلاعات قبلی دارید؟" را کلیک کنید
2. فایل پشتیبان را انتخاب کنید
3. سیستم خودکار نسخه و سازگاری را بررسی می‌کند

## نصب مجدد (Fresh Install)

### روش ۱: از تنظیمات
1. به تنظیمات > پشتیبان‌گیری بروید
2. دکمه «خروجی (Export)» → فرمت و محدوده را انتخاب کنید → ذخیره
3. دکمه «حذف کامل دیتابیس» را بزنید
4. در مرحله ۱: پشتیبان ذخیره کنید
5. در مرحله ۲: تأیید نهایی حذف
6. برنامه مجدداً راه‌اندازی می‌شود
7. برای بازیابی: دکمه «واردات (Import)» → فایل را انتخاب کنید

### روش ۲: دستی
1. فایل `pos.db` را از مسیر بالا حذف کنید
2. پوشه `backups/` را حذف کنید
3. برنامه را مجدداً اجرا کنید

## ارتقا نسخه
هنگام ارتقا از نسخه قدیمی به جدید، دیتابیس خودکار مهاجرت می‌شود:
- ستون‌های جدید با مقدار پیش‌فرض اضافه می‌شوند
- جداول جدید ایجاد می‌شوند
- نیازی به حذف دیتابیس نیست

## فرمت‌های خروجی
| فرمت | توضیح | مناسب برای |
|------|-------|-----------|
| SQLite (.db) | کپی مستقیم دیتابیس | بازیابی سریع، جابجایی بین دستگاه‌ها |
| JSON (.json) | خروجی متنی با متادیتا | مطالعه، ویرایش، انتقال بین نسخه‌ها |
